### 双通道通信协议（新）

**重要：本项目使用 `sessions_send` + `message` 双通道通信机制**

#### 通信流程

每个任务流转包含两个动作：

1. **内部通信**（`sessions_send`）
   - 目标：通知下一个 agent 开始工作
   - 方式：使用 `sessions_send` 工具发送给目标 agent
   - 内容：任务详情、上下文、要求

2. **频道确认**（`message`）
   - 目标：让老板 <@1227127547477753967> 知道工作流正在推进
   - 方式：在项目频道发送确认消息
   - 内容：简洁明了，说明谁收到了什么任务

#### 消息模板

**任务流转确认（每次传递时发送）：**
```
<@1227127547477753967> 📨 任务流转
[发送者] ([角色]) → [接收者] ([角色])
任务：[任务描述]
状态：已通知，等待执行
```

**进度汇报（关键节点发送）：**
```
<@1227127547477753967> 📊 [执行者] 进度汇报
阶段：[当前阶段]
状态：[进行中/已完成]
成果：[关键产出]
下一步：[下一步动作]
```

**开始执行确认（收到任务后发送）：**
```
<@1227127547477753967> ✅ [接收者] 已收到任务
来自：[发送者]
任务：[任务描述]
状态：开始执行
```

### 自动化规则

**关键词触发：**
- "PRD已完成" → 通知老板审批
- "批准" → 通知八阿哥写测试用例
- "测试用例已完成" → 通知混子评审
- "测试用例确认通过" → 混子开始开发
- "开发已完成" → 通知八阿哥测试
- "测试通过" → 通知摸总验收
- "验收通过" → 通知亏总结复盘

**争议处理：**
- 讨论超过5轮 → 自动上报老板
- 单轮超过10条消息/5000字 → 自动上报
- 每轮间隔2小时冷却期

**无响应处理：**
- 24小时无响应 → 提醒当事人
- 48小时无响应 → 上报老板

### 关键审核点

1. **PRD审批**（必须老板批准）
2. **测试用例评审**（八阿哥+混子技术自治）
3. **争议决策**（无法达成一致时老板拍板）
4. **双通道通信**（每次任务流转必须同时发送 sessions_send + 频道确认消息）
